ROLE
You are a senior full-stack engineer. Build a local web app MVP: “Skill Search & Stack Builder”.
Goal: make 631+ skills searchable and understandable, with an exportable recommended stack.

HARD CONSTRAINTS
- Local-first. No external services required.
- Use SQLite with FTS5 for search (BM25 ranking).
- Parse the existing skills catalog file: /mnt/data/CATALOG.md (copy it into the repo at ./data/CATALOG.md if needed).
- Deterministic behavior: explain ranking and show “why this result” signals.
- Keep it simple: minimal dependencies, clean code, predictable folder structure.
- No auth. No analytics. No network calls required.

DELIVERABLES
1) A runnable web UI (local):
   - Search bar
   - Filters: Category, Tags
   - Results list with score + matched fields
   - Skill detail page (“Skill Card”)
   - “Add to Stack” selection panel
   - Export button: generates .agent/skill_stack.md (download + optional write to repo if permitted)
2) Data pipeline:
   - Parse CATALOG.md into normalized records
   - Import into SQLite + FTS index
   - Provide API endpoints for search, skill detail, tags/categories, stack export
3) Docs:
   - README with setup/run instructions
   - Clear commands: init DB, run dev server, build

TECH STACK (choose and implement)
- Backend: Node.js + Express (or Fastify)
- DB: SQLite (better-sqlite3) with FTS5
- Frontend: Next.js (App Router) OR Vite + React
Pick one; prioritize speed of MVP and simplicity (Vite+React is acceptable). Use TypeScript.

APP FEATURES (MVP SPEC)
A) Search
- Input: free text query
- Use SQLite FTS5 over:
  - skill name
  - description
  - triggers
  - tags (as text)
- Return results ordered by BM25.
- Show:
  - skill name
  - category
  - top tags
  - score
  - matched fields (e.g. “matched: triggers, description”)
- If query is empty: show “top skills” (recently viewed or alphabetic) with filters.

B) Filters
- Category dropdown (distinct categories)
- Tags multi-select (top tags + search within tags)
- “Only show skills with Skill Card enriched fields” (toggle; optional)

C) Skill Detail (“Skill Card”)
Display:
- Name, Category, Description, Tags, Triggers (from catalog)
- PLUS editable fields (stored separately in DB table skill_cards):
  - use_when (bullets)
  - avoid_when (bullets)
  - inputs (text)
  - outputs (text)
  - activation (text)
  - similar (list of skill IDs)
  - diff_notes (text: “choose A vs B”)
  - risk_level (low/med/high)
Provide an “Edit” mode to update these fields locally (persist to DB).

D) Stack Builder
- Side panel: selected skills
- Remove/reorder
- Export:
  - Generates markdown: .agent/skill_stack.md
  - Include sections:
    - Core (manual selection)
    - Notes (auto: why selected)
    - Activation order (scan → validate → plan → mutate) as a template
    - STOP conditions template (empty placeholders allowed)
- Also export JSON (stack.json) for programmatic use.

E) Explainability (“Why this result”)
On results and detail, show:
- FTS rank score
- Which columns matched (name/description/triggers/tags)
- Matched tokens (best-effort via highlighting)

DATABASE DESIGN
Create SQLite database at ./data/skills.db with tables:

1) skills
- id INTEGER PRIMARY KEY
- name TEXT NOT NULL
- category TEXT NOT NULL
- description TEXT
- tags_json TEXT  (JSON array)
- triggers_json TEXT (JSON array)

2) skill_cards
- skill_id INTEGER PRIMARY KEY REFERENCES skills(id)
- use_when_json TEXT
- avoid_when_json TEXT
- inputs TEXT
- outputs TEXT
- activation TEXT
- similar_json TEXT
- diff_notes TEXT
- risk_level TEXT

3) FTS virtual table
- skills_fts(name, description, triggers, tags, content='skills', content_rowid='id')
Populate triggers+tags as flattened strings.

Provide migration/init script: ./scripts/init_db.ts

PARSING
Write a robust parser for CATALOG.md:
- It contains category sections and markdown tables.
- Extract: category, skill name, description, tags, triggers.
- Tags/triggers may be comma-separated or list-like; normalize to arrays.
- Create stable IDs from insertion order (or hash). Prefer numeric IDs from DB.
